# Theragen Project Planner — Admin & Operations Map

> Generated by `tools/build_admin_map.py` at `13d6317` (2026-06-17) · platform **v2.10** · 21 authoring surfaces

The platform is **data-driven**: every authoring surface is a SharePoint List that the morning sync validates, airlocks, audits, and lands in Azure PostgreSQL, exposed through `bi.*` views to the Power BI model. This map links each List to its full lineage.

```
SharePoint List  ──►  sync (Graph read → validate → airlock → audit)  ──►  Azure PostgreSQL
   (the form)            sync_intake.py / sync_artifacts.py              (pmbok.* / doc_mgmt.*)
                                                                                │
   Power BI report  ◄──  semantic model (TMDL)  ◄──  bi.* views  ◄─────────────┘
   + RLS (Report Access → Scoped Viewer role)
```

## Security & operations

- **Row-level security:** the **Report Access** List → `bi.report_access` drives the **`Scoped Viewer`** role (default-deny; a user sees the union of their active Project / Department / All grants, matched on `LOWER(USERPRINCIPALNAME())`). **`All Access`** is the admin bypass. RLS only takes effect under per-user identities in the Service.
- **Audit trail:** governance state changes write an append-only `doc_mgmt.audit_trail_entry` row in the same transaction (the **Audit** column below lists each surface's actions).
- **Health check:** `tools/governance_health.py` produces an exceptions digest (see [operations docs](../README.md)).
- **Change control:** every change to this platform follows [the SOP](change-control-process.md); the data dictionary ([data-dictionary.md](data-dictionary.md)) and glossary ([glossary.md](glossary.md)) document the model.
